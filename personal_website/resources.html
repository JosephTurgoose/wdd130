Game Engines

Game engines are what game developers use to create their games. I have worked with and love 3 unique game engines: Construct 3, Unreal Engine, and Unity.

Construct 3

Construct 3 is a relatively new game engine optimized for the creation of 2-dimensional games. It is perfect for anyone who is learning the fundamentals of game-development, or anyone dedicated to a 2D game or project they want to creat.

Upsides: Construct 3 is simple to learn and ever-growing

Downsides: Construct 3 is a subscription-based game-engine, meaning almost nothing can be done in Construct without paying money. It is also somewhat limited in complex capabilities and organization due to its relative newness.

Link: https://www.construct.net/en

Unreal Engine 4 and 5

Unreal Engine has made some of the most popular games today. Widely known for it's immense 3D capabilities, Unreal Engine is one of the leading engines in the world for video game production.

Numerous in-depth tutorials for both beginners and experts, active community, monthly free assets, free to use, relatively beginner-friendly.

Learning the coding system from a beginner point can be difficult and large unique projects require time and knowledge of other programs.

Link: https://www.unrealengine.com/en-US/

Unity Engine

Unity is a widely-known free game engine with both 2D and 3D capabilities, although it is most known for its 2D capabilities. In my opinion, this is the 2D Unreal Engine.

Upsides: Easy to learn the basics, supports multiple coding language (though mainly C#), great variety in capabilities, free to use, occasional free assets.

Downsides: Somewhat buggy in places, file corruption issues, somewhat limited 3D capabilities, less beginner support than Unreal Engine.

Link: https://unity.com/
